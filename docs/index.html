Where do I find the examples of the MMA course math in use?

Most of the math, except for the planar and spacial curves is in the Unity Project you can download from this repository. You can also watch the associated videos, that explain a bit about the usage of the maths in medialogy contexts.
The Planar and Spacial Curves maths is done here on this site, which is part of the same repository as the Unity projects.

Planar and Spacial Curves

For the planar and spacial curves, I decided to move outside of Unity. This is to show that the maths are more broadly applicable than just within real-time development engines.
I decided to make a website, as some of you may end up becoming graphical designers for a company, or end up working with front-end UI in some other regard. Typically, pictures you see on different websites are saved as png, jpg, webp, or other similar formats. These formats are all pixel-based, which means that the graphics do not scale well. Many companies therefore design their logos and other elements of their sites using vector-based graphics, often saved as .svg files (Scaleable Vector Graphics). While these files allow for graphics that can be scaled up an infinite amount and still look sharp, they do not allow for animations.
However! Given the that you as a medialogist know both graphical design and coding, you can design a vector-based logo for a company and animate it using code.

The following is an example of a simple bezier-curve with a ball rolling between the two ends of the curve. I've chosen not to make this example anymore complicated, as the web dev environment likely is completely new to most of you, but you can animate functionally anything in any which way you'd like if you understand the maths well enough.

The maths for this animation uses a time measuring variable t which is translated into a 0-1 range. 0 is when the ball is at the top left of the curve, 1 at the top right. Based on this t value, a point on the quadratic Bézier curve, the white curve in the above animation, is computed. r ( t ) = ( 1 - t ) [ ( 1 - t ) P 0 + t P 1 ] + t [ ( 1 - t ) P 1 + t P 2 ] ,  0 ≤ t ≤ 1 If we animate our ball based on that, it will just move inside the curve:

To have it roll on top of the curve, we'll need to calculate the normal vector of the curve. The equation for this is: N ^ ( t ) = B ^ ( t ) × T ^ ( t ) We currently only have the r ( t ) equation, so we're gonna need to figure out how to get the unit tangent vector T ^ ( t ) and the unit binormal vector B ^ ( t ) functions.
Luckily enough, we know T ^ ( t ) = v ( t ) v ( t ) meaning that we know the unit tangent vector is equivalent to the velocity divided by the speed. We also know that v ( t ) = r' ( t ) meaning that the velocity is the derivative of the position, and that v ( t ) = | v ( t ) | meaning that the speed is the length of the velocity. When we have these equations, we can compute T ^ ( t ) the unit tangent vector. v ( t ) = d d t r ( t ) = 2 ( 1 - t ) ( P 1 - P 0 ) + 2 t ( P 2 - P 1 ) Using Pythagoras a 2 + b 2 = c 2 we can compute the length of the velocity vector v ( t ) = | v ( t ) | and get the speed. This gets us all the variables needed for computing the unit tangent vector. T ^ ( t ) = v ( t ) v ( t ) The binormal is computed by taking the cross product of the velocity and acceleration, and the unit binormal is that vector divided by its length B ^ ( t ) = v ( t ) × a ( t ) | v ( t ) × a ( t ) | (this divisiong gives it a resulting length of 1, all unit vectors share this attribute of having a length of 1).

We'll first need to compute the acceleration, which we luckily already know how to find: a ( t ) = v' ( t ) = r'' ( t ) this meaning it's the derivative of the velocity. a ( t ) = d d t v ( t ) = 2 ( P 2 - 2 P 1 + P 0 ) We'll then need to find the cross product of the velocity and acceleration to get the binormal. Using column vectors we can represent the cross product equation as follows: B ( t ) = v ( t ) × a ( t ) = [ B 1 B 2 B 3 ] = [ v 2 a 3 - v 3 a 2 v 3 a 1 - v 1 a 3 v 1 a 2 - v 2 a 1 ] Then we can divide the binormal vector by its length to get the unit binormal vector: B ^ ( t ) = B ( t ) | B ( t ) | Cross products are defined only in three-dimensional space, but we are only working with two dimensions. As such, we'll set the last dimension to be equal to 0. Doing this gets us: [ B 1 B 2 B 3 ] = [ 0 v 2 - 0 a 2 0 a 1 - 0 v 1 v 1 a 2 - v 2 a 1 ] = [ 0 0 v 1 a 2 - v 2 a 1 ] This means that for our two-dimensional situation, the binormal always points in the third dimension with 0 on both of the first dimensions, and that the unit binormal is always equal to: B ^ ( t ) = [ 0 0 1 ]

With both the unit tangent vector and the unit binormal vector found, we can finally compute the unit normal vector of the curve. N ^ ( t ) = B ^ ( t ) × T ^ ( t ) Let's add this to the circle's position and see if that makes it roll on the curve, instead of in the curve.

This is barely different from before, but that's because the unit normal vector only has a length of 1. Let's multiply that length by our circles radius plus the stroke width to have the circle be moved the correct distance from the curve. 

[...]

// Get the normal vector for this point
var [normalX, normalY] = bezierNormal(startX, controlX, endX, startY, controlY, endY, currentLength);

// Multiply the normal vector by the offset we want
var offsetX = normalX * (circleRadius + strokeWidth);
var offsetY = normalY * (circleRadius + strokeWidth);

// Add the point and offset together to get an offset point
targetX += offsetX;
targetY += offsetY;

[...]

Now our circle rolls on the curve. I would personally like for it to roll on the other side of the curve though, which can be easily fixed by just multiplying the normal vector by -1. 

[...]

// Get the normal vector for this point
var [normalX, normalY] = bezierNormal(startX, controlX, endX, startY, controlY, endY, currentLength);

// The normal is the opposite direction of the direction we desire
normalX *= -1;
normalY *= -1;

// Multiply the normal vector by the offset we want
var offsetX = normalX * (circleRadius + strokeWidth);
var offsetY = normalY * (circleRadius + strokeWidth);

[...]

We now have the circle roll on the inside of the Bézier curve and take two seconds to go from one end to another.
